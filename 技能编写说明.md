# 色色地牢 - 技能编写说明（给 AI 用）

本文档说明如何在 `界面/app.js` 中**只通过数据与既有函数**编写技能，**尽量复用现有逻辑**，避免重复造轮子。

---

## 一、技能类型与数据结构

角色数据中技能分三类，由结构区分：

| 类型           | 存放位置                            | 说明                                  |
| -------------- | ----------------------------------- | ------------------------------------- |
| **被动**       | `ch.passiveSkills[]`                | 无等级、无 AP，仅展示描述             |
| **基础技能**   | `ch.skills[]` 且 `s.basic === true` | 如攻击、防御，无等级，有 AP           |
| **可升级技能** | `ch.skills[]` 且 `s.basic !== true` | 有等级(Lv1~Lv4 等)、可分支进阶、有 AP |

### 1. 被动技能 `passiveSkills[]`

每项：`{ name: string, effect: string }`。

- `effect` 中可使用**属性占位符**和 **【buff名】**，见下文。
- 界面会用 `resolveSkillEffect(rawEffect, ch)` 替换占位符，再用 `wrapBuffRefs()` 渲染【buff】为蓝色可悬停。

示例：

```json
{ "name": "狼之本能", "effect": "达芙妮每次造成暴击时，回复 [Str × 0.2] 的生命值。" }
```

### 2. 基础技能（攻击/防御等）

在 `ch.skills[]` 中：`{ name, ap, effect, basic: true }`。

- **必须**带 `basic: true`。
- `effect` 写法同被动，用 `[Str × 1]`、`[Def × 1]` 等。

示例：

```json
{ "name": "攻击", "ap": 1, "effect": "对所选单位造成 [Str × 1] 点伤害", "basic": true },
{ "name": "防御", "ap": 1, "effect": "对自身给予 [Def × 1] 点护盾，持续到自己的下个回合开始", "basic": true }
```

### 3. 可升级技能（含分支进阶）

在 `ch.skills[]` 中，**不带** `basic` 或 `basic: false`。可选字段：

| 字段                        | 类型     | 说明                                                                                                                                                                                       |
| --------------------------- | -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `name`                      | string   | 技能名；若已进阶，界面显示为「name-分支name」                                                                                                                                              |
| `ap`                        | number   | 消耗 AP                                                                                                                                                                                    |
| `tags`                      | string   | 可选，如 `"物理/近战/群体"`，显示在名称下方                                                                                                                                                |
| `effect`                    | string   | 无 `effectByLevel` 时使用，支持占位符与【buff】                                                                                                                                            |
| `effectByLevel`             | string[] | **按等级取描述**，如 Lv1 用 `[0]`、Lv4 用 `[3]`；与 `effect` 二选一                                                                                                                        |
| `level`                     | number   | 当前等级，默认 1；升级用技能点                                                                                                                                                             |
| `advancement`               | string   | 已选分支的 id，如 `"A"`；无则未进阶                                                                                                                                                        |
| `advancementReplacesEffect` | boolean  | 可选。为 `true` 时，**进阶后只显示分支效果**，不显示原等级描述、也不在下方追加一段分支描述（相当于分支“替换”主描述）。不设或 `false` 时，进阶后主描述保留 Lv4 等原文，下方再显示分支效果。 |
| `advancementOptions`        | array    | 分支列表，见下表                                                                                                                                                                           |

`advancementOptions` 每项：

| 字段     | 类型   | 说明                                            |
| -------- | ------ | ----------------------------------------------- |
| `id`     | string | 唯一 id，选定后写入 `skill.advancement`         |
| `name`   | string | 分支名，进阶后标题变为「原技能名-name」         |
| `effect` | string | 分支效果描述，**支持** `[Str × 0.2]` 与【buff】 |

示例（狼式旋风）：

```json
{
  "name": "狼式旋风",
  "ap": 1,
  "tags": "物理/近战/群体",
  "effectByLevel": [
    "挥舞巨剑横扫。对敌方全体造成 [Str × 0.6] 的物理伤害。",
    "挥舞巨剑横扫。对敌方全体造成 [Str × 0.7] 的物理伤害。",
    "挥舞巨剑横扫。对敌方全体造成 [Str × 0.8] 的物理伤害。",
    "挥舞巨剑横扫。对敌方全体造成 [Str × 0.9] 的物理伤害。"
  ],
  "advancementOptions": [
    {
      "id": "A",
      "name": "血腥旋风",
      "effect": "命中的每个敌人施加 [Str × 0.2] 层【重伤】，如暴击额外施加 [Str × 0.2] 层【重伤】"
    },
    {
      "id": "B",
      "name": "重点打击",
      "effect": "挥舞巨剑横扫。对敌方全体造成 [Str × 0.9] 的物理伤害。对主要目标造成150%的伤害，若场上仅有1个敌人，伤害提升至200%。"
    }
  ]
}
```

- 有 `effectByLevel` 时，**不要**再写 `effect`，等级描述由 `getSkillEffectForLevel(skill, lv)` 按等级取对应句。
- 进阶后界面会：显示「狼式旋风-血腥旋风」、等级徽标显示 **Lv5**、下方多一段分支效果描述；分支描述会随属性加减**实时重算**（见后文）。若技能带有 `advancementReplacesEffect: true`（如「剑脊格挡」），则进阶后**只显示分支效果**，不显示原 Lv1–Lv4 描述、也不在下方再挂一段分支文。

## 二、必须复用的机制（不要自己实现）

### 1. 属性占位符

**支持的格式**：
- 单项：`[Str × 0.2]`、`[Def × 1]` 等。
- 多项（同一方括号内用 `+` 连接，每项向下取整后相加）：`[Def × 0.8 + Str × 0.4]`、`[Def × 1.5 + Str × 0.6]`。

注意：属性名用英文（Str/Agi/Int/Sta/Def），乘号为 ×，方括号为半角 `[]`。

- **解析**：由 `resolveSkillEffect(effect, ch)` 或 `resolveSkillEffectWithStats(effect, displayStats)` 统一替换：先对每个「属性×系数」向下取整，再对多项求和。
- **不要**在描述里写「str*0.2」或自定义格式；用上述占位符即可，界面会显示为计算后的数字（且随属性点变化更新）。

### 2. 技能描述按等级取文案

- 可升级技能若提供 `effectByLevel` 数组，**必须**通过现有函数取当前等级描述：
  - `getSkillEffectForLevel(skill, lv)`：返回 `skill.effectByLevel[lv - 1]`（或首项），不要手写按等级取字符串的逻辑。

### 3. Buff 名与悬浮说明（【buff名】）

- 在**任意** `effect` / 分支 `effect` 中写 **【buff名】**（全角括号 + 与 `BUFF_DEFINITIONS` 中 `id`/`name` 一致）。
- 界面会自动：
  - 将【buff名】渲染为蓝色可点击/悬停的 span；
  - 悬停或点击时用 `getBuffTooltipText(buffId)` 或战斗内 `getBuffEffectTooltip(buffId, layers)` 显示说明，均读取 `BUFF_DEFINITIONS` 的 `desc` 字段。
- **不要**自己拼 HTML 或再做一层「buff 高亮」；只要在描述里写【重伤】、【燃烧】等即可。

### 4. 新增 Buff 定义

- 所有 buff 定义在 **`BUFF_DEFINITIONS`** 中，载入游戏时会注入到聊天变量，供规则/LLM 使用。
- 新增 buff 时**只追加**一项，例如：
  - `{ id: '重伤', name: '重伤', rule: '...', desc: '回合结束时受到等同于层数的伤害' }`
- `rule` 用于规则/LLM；`desc` 用于界面悬浮/说明显示，所有 buff 显示均读此字段。

### 5. 属性与技能点常数

- 属性点：`FREE_POINTS_PER_LEVEL = 10`，每级 (level-1)*10 点可分配；五维为 `str, agi, int, sta, def`，对应 `BONUS_KEYS`（如 `bonusStr`）。
- 技能点：`SKILL_POINTS_PER_LEVEL = 3`，每级 3 点，用于升级或分支进阶；已用点数在 `ch.skillPointsSpent`。
- 计算「当前显示属性」用 `getDisplayStat(ch, 'str')` 等（已含 bonus）；计算未用属性点用 `getUnspentPoints(ch)`，未用技能点用 `getUnspentSkillPoints(ch)`。**不要**自己再算一遍点数公式。

### 6. 角色技能等级与进阶状态

- 可升级技能的当前等级在 `skill.level`（默认 1）；升级用技能点，由界面加减号与「确定」写入。
- 分支进阶：用户选择分支后，只设置 `skill.advancement = 选项id`，并增加 `ch.skillPointsSpent`；**不要**在 `passiveSkills` 或别处再复制一份分支效果，界面会根据 `advancement` + `advancementOptions` 在同一张技能卡内显示分支描述。

---

## 三、界面已做的「实时更新」

- 角色详情里加减属性时，会调用 `refreshSkillDescriptions()`，其中会：
  - 用 `resolveSkillEffectWithStats(rawEffect, displayStats)` 更新主技能描述（`.skill-desc`）；
  - 用同样 `displayStats` 更新**分支效果**（`.skill-desc-advancement`）的文案。
- 因此分支效果里写的 `[Str × 0.2]` 会随力量等变化**自动变为最新数字**，无需在写技能时做额外逻辑。

---

## 四、写新技能时的检查清单

1. **被动**：只改 `ch.passiveSkills[]`，每条 `{ name, effect }`；`effect` 用 `[Str × 0.2]` 与【buff名】。
2. **基础技能**：在 `ch.skills[]` 里加 `{ name, ap, effect, basic: true }`，占位符同上。
3. **可升级技能**：
   - 用 `effectByLevel` 时不要写 `effect`；用 `getSkillEffectForLevel(skill, lv)` 取描述。
   - 分支写在 `advancementOptions`，每项 `{ id, name, effect }`；`effect` 支持占位符与【buff】。
   - 进阶后只设 `skill.advancement = id` 和 `skillPointsSpent += 1`，不复制到别处。
4. **新 Buff**：在 `BUFF_DEFINITIONS` 中追加 `{ id, name, rule, desc }`（`desc` 为界面显示用）；描述里用【buff名】即可自动蓝字+悬浮。
5. **数字一律用占位符**：描述中凡依赖力量/敏捷等的地方，写 `[Str × 系数]` 等，不要写死「2层」；系数可小数，界面会向下取整后显示。

按上述方式写技能，即可最大程度复用现有函数与界面行为，保持风格一致且易维护。
